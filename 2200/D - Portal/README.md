<h3><a href="https://codeforces.com/contest/2200/problem/D" target="_blank" rel="noopener noreferrer">Portal</a></h3>

<div class="header"><div class="title">D. Portal</div><div class="time-limit"><div class="property-title">time limit per test</div>2 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p>You are given a permutation$$$^{\text{∗}}$$$ $$$p$$$ of length $$$n$$$. There are also two portals located at positions $$$x$$$ and $$$y$$$ ($$$x \lt y$$$).</p><p>A portal at position $$$i$$$ is initially located between the $$$i$$$-th and $$$(i+1)$$$-th elements of the array. Specifically, if $$$i=0$$$, then the portal is located before the first element of the array, and if $$$i=n$$$, then the portal is located after the last element.</p><p>You may perform either of the following two operations <span class="tex-font-style-bf">as many times</span> as you like: </p><ol> <li> Remove the element to the immediate left of one portal and insert it to the immediate right of the other portal. </li><li> Remove the element to the immediate right of one portal and insert it to the immediate left of the other portal. </li></ol><p>Let $$$\mathbf{\color{red}{\mathcal{O}}}$$$ denote a portal. For example, if $$$p$$$ is $$$[3,\mathbf{\color{red}{\mathcal{O}}},2,4,\mathbf{\color{red}{\mathcal{O}}},1]$$$: </p><ul> <li> Using operation $$$1$$$ on the left and right portals respectively results in the arrays $$$[\mathbf{\color{red}{\mathcal{O}}},2,4,\mathbf{\color{red}{\mathcal{O}}},3,1]$$$ and $$$[3,\mathbf{\color{red}{\mathcal{O}}},4,2,\mathbf{\color{red}{\mathcal{O}}},1]$$$. </li><li> Using operation $$$2$$$ on the left and right portals respectively results in the arrays $$$[3,\mathbf{\color{red}{\mathcal{O}}},4,2,\mathbf{\color{red}{\mathcal{O}}},1]$$$ and $$$[3,1,\mathbf{\color{red}{\mathcal{O}}},2,4,\mathbf{\color{red}{\mathcal{O}}}]$$$. </li></ul><p>Find the lexicographically$$$^{\text{†}}$$$ smallest permutation you can obtain using these operations. Note that portals do not affect the lexicographical comparison of permutations.</p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$A permutation of length $$$n$$$ is an array of length $$$n$$$ containing each integer from $$$1$$$ to $$$n$$$ exactly once.</p><p>$$$^{\text{†}}$$$A permutation $$$a$$$ is lexicographically smaller than permutation $$$b$$$ if there exists an index $$$i$$$ such that $$$a_j = b_j$$$ for all indices $$$1 \leq j  \lt  i$$$ and $$$a_i  \lt  b_i$$$.</p></div></div><div class="input-specification"><div class="section-title">Input</div><p>The first line contains an integer $$$t$$$ ($$$1 \leq t \leq 2\cdot 10^4$$$) — the number of test cases.</p><p>For each test case, the first line contains three integers $$$n$$$, $$$x$$$, and $$$y$$$ ($$$1 \leq n \leq 2 \cdot 10^5$$$, $$$0 \leq x  \lt  y \leq n$$$).</p><p>The second line of each test case contains $$$n$$$ integers $$$p_1, p_2, \dots, p_n$$$ — a permutation of length $$$n$$$.</p><p>The sum of $$$n$$$ over all test cases does not exceed $$$2 \cdot 10^5$$$.</p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, output a line with $$$n$$$ integers — the lexicographically smallest permutation you can obtain.</p></div><div class="sample-tests"><div class="section-title">Example</div><div class="sample-test"><div class="input"><div
